==============================================================
PRACTICE PROTOCOL  (per problem)
==============================================================

  1. READ the problem twice. Restate it in your own words.
  2. WRITE 1-2 example inputs/outputs on paper.
  3. THINK aloud for up to 5 minutes - no code yet.
  4. PICK a brute-force approach. State its time complexity.
  5. CODE the brute force, run mentally on the examples.
  6. OPTIMIZE only if time allows.
  7. TEST edge cases: empty input, single item, max size, duplicates.
  8. EXPLAIN final solution as if presenting to the interviewer.

If stuck past 20 minutes -> read the hint, NOT the full solution.
If still stuck after hint -> read solution, mark for retry tomorrow.
